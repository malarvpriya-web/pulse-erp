## BACKEND AUDIT

### Repository: inventoryItem.repository.js

**CREATE (create method):**
```javascript
const { item_code, item_name, item_type, unit_of_measure, reorder_level, 
        standard_cost, inventory_account_id, expense_account_id, description, company_id } = data;
```
When frontend sends `{sku, name, category, unit, ...}`:
- `item_code` ← auto-generated by `getNextCode()` ✅
- `item_name` ← undefined (frontend sends `name`) ❌ CREATES NULL NAME
- `item_type` ← undefined ❌
- `unit_of_measure` ← undefined (frontend sends `unit`) ❌
- `standard_cost` ← undefined (frontend sends `unit_price`) ❌

**UPDATE (update method):**
```javascript
const { item_name, reorder_level, standard_cost, description, is_active } = data;
UPDATE inventory_items SET item_name = $1 ...
```
When frontend sends `name` instead of `item_name`:
- `item_name` ← DB spread in `openEdit()` adds `item_name` via `...item`, so edit mode works
- BUT `name` (empty string from emptyForm()) also in payload — DB column not updated with it
- Net effect in EDIT: item_name preserved from DB spread ✅ BUT unit, category changes are NOT saved
